# MiMo TTS 语音合成技能

## 触发条件
当用户要求：
- 发送语音消息
- 朗读文本
- TTS 合成
- 语音回复
- 克隆声音
- 设计音色
- 唱歌

## 模型选择

| 模型 | 功能 | 特点 |
|------|------|------|
| `mimo-v2.5-tts` | 使用预置精品音色进行语音合成 | 支持唱歌，音色固定 ✅ 默认 |
| `mimo-v2.5-tts-voicedesign` | 通过文本描述定制音色 | 通过文本描述自动生成音色，无需预置或音频样本 |
| `mimo-v2.5-tts-voiceclone` | 基于音频样本复刻任意音色 | 通过音频样本精准复刻音色，实现任意声音的语音合成 |

## 音频标签

### 基本用法
在文本前添加 `(标签)` 格式即可控制发音风格，例如：
```
(唱歌)原谅我这一生不羁放纵爱自由
(怅然)这么多年过去了，再走过那条街
(东北话)哎呀妈呀，这天儿也忒冷了
```

### 支持的标签类型

**基础情绪**：开心、悲伤、愤怒、恐惧、惊讶、兴奋、委屈、平静、冷漠、压抑的愤怒、带着哽咽的笑意、温柔但疲惫、狂躁中的温柔

**复合情绪**：怅然、欣慰、无奈、愧疚、释然、嫉妒、厌倦、忐忑、动情

**整体语调**：温柔、高冷、活泼、严肃、慵懒、俏皮、深沉、干练、凌厉

**音色定位**：磁性、醇厚、清亮、空灵、稚嫩、苍老、甜美、沙哑、醇雅

**人设腔调**：夹子音、御姐音、正太音、大叔音、台湾腔

**方言**：东北话、四川话、河南话、粤语

**角色扮演**：孙悟空、林黛玉

**语速与节奏**：吸气、深呼吸、叹气、长叹一口气、喘息、屏息

**情绪状态**：紧张、害怕、激动、疲惫、委屈、撒娇、心虚、震惊、不耐烦

**语音特征**：颤抖、声音颤抖、变调、破音、鼻音、气声、沙哑

**哭笑表达**：笑、轻笑、大笑、冷笑、抽泣、呜咽、哽咽、嚎啕大哭

**唱歌**：唱歌、sing、singing

## 音色设计模式 (VoiceDesign)

### 触发方式（三选一）：

1. **`[voice: ...]` 前缀语法（推荐，通过 TTS provider 自动路由）：**
   ```
   [voice:一位年迈的老先生，嗓音沙哑沧桑]各位听众好。
   ```
   - 描述是已知预置音色名（如 `茉莉`、`mia`）→ 使用预置音色，不触发 VoiceDesign
   - 描述是自定义文本 → 自动切换到 `mimo-v2.5-tts-voicedesign` 模型

2. **`--design` 命令行标志（直接调用脚本时）：**
   ```bash
   python mimo_tts.py <output_path> <text> --design
   ```

3. **`MIMO_TTS_USE_DESIGN=1` 环境变量**

⚠️ **TTS provider 命令只传 `{output_path} {text}`，不传额外参数。所以 `[voice: ...]` 前缀是通过 Hermes TTS 工具使用 VoiceDesign 的唯一方式。**

### 调用示例（直接脚本）：
```bash
python ~/AppData/Local/hermes/scripts/mimo_tts.py <output_path> <text> [voice_desc] --design
```

### 如何写好音色描述

描述越具体、越生动，生成的音色越贴近预期。

#### 关键维度：
| 维度 | 示例 |
|------|------|
| 性别与年龄 | "young woman in her mid-20s"、"五十多岁的中年男性" |
| 音色/质感 | "deep and gravelly"、"丝滑醇厚、带着磁性" |
| 情绪/语气 | "warm and confident"、"温柔但带着一丝疲惫" |
| 语速/节奏 | "slow and deliberate"、"语速极快，像连珠炮" |

#### 可选维度（增加丰富度）：
- 角色/人设：narrator, podcast host, 评书先生, 深夜电台DJ
- 说话风格：casual and colloquial, 一本正经地, 压低嗓音像在密谋
- 场景描写：narrating a nature documentary, 在给投资人路演
- 年代参照：1940s film noir, 八十年代译制片配音

#### 写法建议：
1. **简洁描述型** — 用关键词或一句话快速勾勒声音轮廓
   ```
   Heavy Russian accent, gruff middle-aged male, blunt and matter-of-fact.
   ```

2. **专业描述型** — 通过场景、人设或多维度细节立体刻画声音
   ```
   Young female, extreme close-up with a binaural, ear-to-ear ASMR feel. 
   Audible breathing, subtle swallowing, and soft natural lip sounds. 
   She speaks very slowly, creating a deeply relaxing and immersive experience.
   ```
   ```
   一位年迈的老先生，说带北方口音的普通话，语速缓慢而沉稳，嗓音略带沙哑和沧桑感，
   仿佛一位饱经风霜的老爷爷在讲故事，充满岁月的智慧。
   ```

#### 注意事项：
- 长度：1-4 句即可，核心特征描述清楚比堆砌维度更重要
- 避免冲突：不要同时要求矛盾的特征（如"稚嫩的童声 + CEO气场"）
- 避免音质效果词：不要写混响、回声、EQ、压缩等后期处理相关描述
- 避免模糊词：不要用"普通的""正常的""外国的"等缺乏具体指向的描述
- 中英文均可：模型同时支持中英文音色描述

#### 合成文本要贴合音色：
assistant 消息中的合成文本应与音色描述相匹配。例如为"温柔治愈系女声"搭配一段晚安独白，而非一段激烈的体育解说。

#### 场景音色示例：
```bash
# 甜美客服音
python script.ogg "您好，请问有什么可以帮您？" "用温柔甜美的客服小姐姐声音说话" --design

# 严肃新闻音
python script.ogg "本台消息……" "用严肃权威的新闻播音员声音说话" --design

# 可爱动漫音
python script.ogg "主人你好呀！" "用活泼可爱的二次元萌妹声音说话" --design

# 磁性男声
python script.ogg "晚上好，欢迎收听今晚的夜读" "用低沉磁性的成熟男性声音说话" --design

# ASMR助眠音
python script.ogg "闭上眼睛，慢慢放松……" "Young female, ASMR feel, very slow and soft, creating a relaxing experience" --design

# 深夜电台
python script.ogg "夜深了，让我们聊聊心事" "深夜电台DJ，低沉温柔的男声，语速缓慢，带着治愈感" --design

# 评书先生
python script.ogg "话说那天……" "评书先生，中年男性，京腔，抑扬顿挫，节奏感强" --design

# 童声
python script.ogg "小朋友们好！" "稚嫩可爱的童声，五六岁小孩，天真活泼" --design
```

## 音色克隆模式 (VoiceClone)

### 触发方式：

1. **`--clone` 命令行标志 + 音频文件（直接调用脚本时）：**
   ```bash
   python mimo_tts.py <output_path> <text> --clone <path_to_audio_sample.mp3>
   ```

2. **VoiceClone + 风格控制：**
   ```bash
   python mimo_tts.py <output_path> <text> "用温柔的语气说" --clone <path_to_audio_sample.mp3>
   ```

### 音频样本要求：
- 格式：支持 MP3 或 WAV
- 长度：建议 5-30 秒
- 质量：清晰无背景噪音
- 内容：自然说话的语音样本

## 最佳实践

### 1. 音色选择优先级
1. 用户明确指定 → 使用指定音色
2. 语境匹配 → 根据场景选择
3. 语言匹配 → 中文/英文默认音色
4. 兜底 → `茉莉`（中文）或 `Mia`（英文）

### 2. 语气选择优先级
1. 用户明确指定语气 → 使用指定语气
2. 内容有明显情感倾向 → 根据情感选择
3. 用户意图明确 → 根据意图映射表选择
4. 兜底 → 自然平稳的语气

### 3. 风格控制原则
- **简单场景**：不添加风格，使用默认自然语调
- **情感场景**：添加情绪描述
- **专业场景**：添加角色和语调描述
- **创意场景**：使用复合情绪和详细描述

### 4. 文本处理
- 移除 markdown 格式（**、*、` 等）
- 移除代码块
- 保留标点符号（有助于节奏）
- 长文本可分段合成

### 5. 输出格式
- Telegram 语音：`.ogg` (Opus)
- 本地播放：`.mp3` 或 `.wav`
- 高质量保存：`.wav`

### 一键生成命令

```bash
TTS_TMP="${TMPDIR:-${TEMP:-${TMP:-/tmp}}}"
```

### ⚠️ 关键陷阱

- **MEDIA 路径必须用绝对路径**，相对路径会报 "file not found"
- **不要用相对路径 `media/`** — hermes 根目录下没有 `media/` 目录，ffmpeg 会静默失败（脚本不检查返回码）。必须用 `$TMPDIR` / `$TEMP` / `/tmp` 或 `~/AppData/Local/hermes/audio_cache/`
- **不要用中文解释流程**，用户会觉得太慢
- **不要问"要发语音吗？"**，直接生成发送
- **VoiceDesign 默认描述陷阱** — 脚本默认音色描述是"温柔甜美的年轻女性声音说话"。如果通过 `--design` 但没传描述参数，会生成女声。务必通过 `[voice: ...]` 前缀传入完整描述
- **音频标签要放在文本开头**，不要放在中间或结尾
- **VoiceClone 需要音频样本文件**，确保文件存在且格式正确（mp3 或 wav）

## 注意事项

1. **API 限制**：单次请求最长 5000 字符
2. **响应时间**：通常 1-3 秒
3. **音频质量**：WAV > MP3 > OGG
4. **风格复杂度**：简单指令效果更稳定
5. **唱歌模式**：仅支持中文歌词，使用 `mimo-v2.5-tts` 模型
6. **VoiceDesign 限制**：不支持唱歌模式、预置音色与音色复刻
7. **VoiceClone 限制**：不支持唱歌模式、预置音色与音色设计

## 故障排除

### 问题：音色不匹配
- 检查音色 ID 是否正确
- 确认使用的是中文/英文对应音色

### 问题：风格不生效
- 风格描述放在 `user` 消息中
- 描述尽量具体、自然
- 避免过于复杂的指令

### 问题：音频质量差
- 使用 WAV 格式获取最高质量
- 检查网络连接
- 尝试其他音色

### 问题：VoiceClone 失败
- 检查音频文件格式（支持 mp3 或 wav）
- 确保文件存在
- 音频样本质量要清晰无噪音
- 样本长度建议 5-30 秒

### 问题：HTTP/URL Error
- 检查网络连接
- 确认 API Key 有效
- 查看控制台错误信息
